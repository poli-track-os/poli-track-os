# Security Policy

## Supported versions

Poli Track is currently pre-alpha software. Only the latest code on the default
branch is considered supported for security fixes.

| Version | Supported |
| --- | --- |
| `main` | Yes |
| older commits and forks | No |

## Reporting a vulnerability

- Do not open a public GitHub issue for an undisclosed security problem.
- Preferred path: use GitHub private vulnerability reporting for this repository once it is enabled.
- If private vulnerability reporting is not yet enabled, contact the maintainer before public disclosure and give a reasonable window for investigation and remediation.

## What to include

- A short description of the issue and affected area.
- Reproduction steps or a proof of concept.
- Impact assessment.
- Any suggested mitigation if you already have one.

## Response goals

- Initial triage acknowledgement within 5 business days.
- Status update after reproduction or rejection.
- Public disclosure only after a fix or mitigation is available.
